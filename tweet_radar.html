Twitter Stream Discord Bot

Real-time Tweet Stream to Discord



Inspiration

As a sneaker enthusiast, I follow Twitter accounts that post updates on the latest releases. Popular releases sell-out in minutes, so having to not miss out on releases requires proactively checking or turning on notifications from these twitter accounts.

Since I would only like to keep track of certain releases (ex. Jordan 1s) and notifications from certain accounts can be quite frequent, the idea came up to create a program that filters tweet notifications based on keywords and accounts.

I wanted to use Discord’s real-time notification feature and stream tweets to a discord channel so that I could have the sneaker tweets neatly grouped into a channel for insights and ease of use.

Here is an example of the end result, in which I filtered tweets based on the keyword "restock" from sneaker accounts.


Background

This project involves streaming filtered tweets to discord channels so that users can stay updated on the latest news for topics of interest.

Using Twitter and Discord APIs, an individual can customize the rules for the tweets they receive and group topics of tweets into channels.

This is a personal project that I currently have running on a remote server so that I can receive the notifications I desire.


Problem Statements

1. Users only want to receive filtered real-time notifications from specific users.

2. Users have many different topics they follow and would like an organized channel system to make research easier


User Stories

The application will benefit anyone who wants to keep track of certain topics in real-time.

Here are some user stories examples:

  • The sneaker enthusiast who wants sneaker release info before the release dates but does not want to be notified on every tweet from accounts they follow.
  • The journalist who is tasked to write an article on a specific topic and wants to receive information from specific sources in real-time.
  • The scientist who is researching a specific topic and wants to follow discussions started by other scientists.
  • The music lover who does not want to miss out concert tickets from their favorite artists but only wants tickets for concerts in their area.
  • Someone who wants their tweets categorized into different topic channels so that it is easier to digest.

    • Brainstorming and Protoyping

    There are 3 pieces of information that can be set to make this program customizable for the user:

  • The name of the topic
  • The rule that defines what tweets will be filtered in the stream
  • The discord channel ID, which specifies what channel the tweets will stream into


    • Design question #1: How can the user modify their rule?

    Answer: After the user edits the rules,json file, they can restart the program and the program will clear all the rules, and then apply the rules and channels reflected in the file.

    Design question #2: If the user wants to have multiple topics, how can they add more?

    Answer: The user can configure multiple rule objects in the rules,json file. After they restart the program, they will have filtered streams for multiple topics!


    Technical Implementation

    The program consists of a Node.js backend script on the server.

    Here are its components:

  • Connection to the Twitter Rule Stream endpoint to set the rules for the data stream
  • Connection to the Twitter Stream endpoint to request a data stream
  • Connection to Discord endpoint to activate the Discord Bot
  • Listen to the stream for any data, and send the messages to the assigned discord channel
