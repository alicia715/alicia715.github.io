Genre Cloud

An Interactive Data Visualization Website

Website Link:

https://genrecloud.azurewebsites.net/

Background

Genre Cloud is a website to discover your favorite genres and artists in an interactive word cloud.

Using listening history from Spotify APIs, Genre Cloud reveals insights about your genre preferences and connects you to new music.

This project is a personal exploratory project. I work on the ideation, design, prototyping, and distribution.

Problem Statements

1. Non-premium Spotify members do not have access to their top genres. Music enthusiastic or people with eclectic music tastes who are non-premium Spotify members are curious about their top genres but do not have access to this information.

2. Premium Spotify members have access to their top genres but the method of how these top genres are determined is not revealed. They are skeptical of the information presented to them by Spotify.

3. There are over 1000 music subgenres. A user may not be familiar with some of their top genres displayed by Spotify and want to conveniently find information about those genres.

User Research

Premium Spotify users found their top music genres in Spotify 2020 Wrapped. I was browsing Reddit and found these comments:

  • "Why is my favorite music genre pop, while never listening to it?" (refers to Problem Statement 2)
  • "Does anybody know how Spotify determines your top played genres?" (refers to Problem Statement 2)

    • There are over 1000 music subgenres recorded to date and even more maintained by Spotify--they use Artificial Intelligence to category new genres and patterns. Some users are confused when they see that their top genre is a genre they are unfamiliar with.

  • "What is 'scorecore?' Apparently it's one of my top genres, but I've never even heard of it before." (refers to Problem Statement 3)
  • "What genre is Escape Room?" (refers to Problem Statement 3)

    • Brainstorming and Protoyping

    Because we are working with data, we can use a data visualization to solve each part of our problem statement. I chose a word cloud because it is a visual representation that draws attention to text. In our case, the words are the genres.

    Visual and interactive features of the visualization will provide insights to users.

    Size: The bigger the genre, the more the user tends to listen to it.

    Hover: Hovering over a genre will provide a user with a list of artists they listen to that fall under that genre. This answers Problem Statement 2 (how user top genres are determined).

    Click: Clicking on a genre will take you to a playlist filled with songs from that genre (special thanks to Glenn McDonald, creator of "The Sounds of Spotify"). This answer Problem Statement 3 (creating a convenient way for users to learn about genres)

    Technical Implementation

    Tools: Flask, Python, Javascript, Spotify API, HTML/CSS, Azure Cloud Services, P5.js

    Flask is a micro web framework for Python for building web applications. The Flask project structure separates the frontend, service, and application for modularization.

    Application module:

  • Defines the URL endpoints and what information is presented at these endpoints (the business logic)
  • Acts in the middle of Service and Frontend modules (similarily to MVC architecture). (1) uses the functions defined in the Service module. (2) performs data binding, so information on the server where the program is running can be presented to the view (frontend) for the user

    • Service module:

  • Implements OAuth 2.0, authentication protocol that allows one application to interact with another without giving away user password. Visually, these are the steps: (1) users login into Spotify (2) Spotify asks if the user will grant the application (Genre Cloud) access to specific user information (3) user agrees (4) Spotify generates a token and takes user to redirect page (5) the user arrives back to the application (Genre Cloud), and Genre Cloud uses the token to access the specific user information the user approved on sharing. The application now functions with this users data!
  • Defines functions to filter the data coming back, and then restructuring the data to get a dictionary in the form, "genre" to "list of artists of that genre".

    • Frontend:

  • Folder structure is static/css, static/js, and templates/html
  • Javascript file handles the user data and fills it into the world cloud
  • Html creates the web pages and CSS styles

    • Fun: Twinkly stars created with P5.js
